## 3. 核心发现：训练与应用错配

### 训练学的是什么？
```
下月收益 ≈ w1×ROE + w2×PE + w3×ADX + ...
```
权重是针对**原始因子值**的线性系数。

### 回测时权重用在了哪里？
```python
combined = fund_score * 0.1094 + growth_score * 0.2996 + tech_score * 0.5911
```
这里的 `fund_score` 是 `score_fundamentals()` 输出的**0-1阈值化分数**，不是原始ROE值。

### 错配本质
- **训练**：原始数值 → 线性预测收益
- **回测**：开关分数（达标/不达标）→ 加权组合

**训练和应用的是两个不兼容的体系。**

## 4. 方案A：维度分数训练

直接用 `fund_score`、`growth_score`、`tech_score` 训练维度权重。

### 关键发现：三个维度全部与下月收益负相关

| 维度 | 与收益相关性 | Q1(最低分)平均收益 | Q5(最高分)平均收益 |
|------|------------|------------------|------------------|
| 基本面 | -0.0014 | +0.00% | -0.02% |
| 成长 | -0.0347 | +0.52% | -0.65% |
| **技术面** | **-0.0807** | **+1.49%** | **-1.30%** |

**技术面分数是强反向指标**——分数越高的股票，下个月亏得越多。

归一化后训练权重：基本面100%，成长0%，技术面0%。

## 5. 方案C：反向技术面权重（核心验证）

将技术面权重设为 **-1**（分数越低越好），基本面和成长设为0：

```python
WEIGHTS = {"fundamentals": 0, "growth": 0, "technical": -1}
```

### 三版本对比

| 指标 | 原策略 (35/40/25) | Ridge (11/30/59) | **反向技术 (0/0/-1)** |
|------|------------------|-----------------|----------------------|
| 年度收益 | -26.83% | -46.57% | **+2.01%** |
| 超额收益 | -5.56% | -25.30% | **+23.28%** |
| Alpha | -0.0532 | -0.3439 | **+0.2282** |
| Beta | 0.9556 | 1.0878 | **0.7492** |
| 夏普比率 | -0.931 | -1.794 | **+0.084** |
| 最大回撤 | -34.89% | -48.31% | **-21.17%** |
| 月度胜率 | 36.4% | 27.3% | **63.6%** |
| 信息比率 | -0.162 | -1.345 | **1.270** |

**反向技术面策略在2022年大幅跑赢基准23.28个百分点**，选股从趋势股转向防御型低趋势股。

## 6. 结论

1. **原策略的评分逻辑在2022年是反向有效的**：`score_technical` 选到的"趋势最强"股票，恰恰是下个月回调最多的
2. **不是权重问题，是评分函数与市场结构不匹配**：A股月度维度存在明显的短期反转效应
3. **训练本身没有bug**，但训练权重被应用在了不兼容的评分体系上
4. **低Beta（0.75）是反向策略超额收益的主要来源**——2022年熊市中，低波动策略本身占便宜
